### Что нужно сделать (3 группы по сложности)

#### A. Доделать недостающее (средняя сложность)

**6. Редактор «Популярных изделий»**
- Добавить в Контент сайта блок «Популярные изделия»: список SKU/ID товаров для слайдера на главной + порядок + вкл/выкл позиции.
- Хранение: `app_settings.key = 'popular_products'`, value `{ items: [{ productId, order, enabled }] }`.
- Компонент `PopularProducts.tsx` тянет этот список и выводит только выбранные товары.

**8. Фон секций**
- Добавить в HomepageEditor поля «Фоновое изображение» для секций: hero (если без видео), popular, categories, advantages, contact.
- Хранение: расширить ключ `homepage` полями `<section>.bgImage`.

#### B. Каталог категории/подкатегории (тяжёлая работа)

**9. Управление деревом каталога из админки**
- Создать таблицу `public.catalog_categories` (id, parent_id, slug, name, image, sort_order, is_active).
- Миграция: добавить таблицу + GRANT + RLS (публичное чтение, запись через admin-api).
- Расширить edge-функцию `admin-api`: операции `categories.list/create/update/delete/reorder`.
- Перевести `CatalogPage`, `categoryOptions` в `AdminPage.tsx`, `dbProducts.ts` (CATEGORY_MAP) на динамическое чтение из БД.
- Миграция данных: сидинг текущих 8 категорий и подкатегорий из `data/products.ts`.
- В админке — новая вкладка «Каталог» (или раздел в «Товары») с CRUD-деревом 2 уровня.

⚠️ Это значительный объём — затрагивает БД, edge-функцию, фронтенд каталога и админку. Полдня-день работы.

#### C. Стилизация блоков (большая работа, спорная польза)

**5. Per-section дизайн (цвет/контур/размер)**
- Это нетривиально: текущий сайт построен на единой дизайн-системе (FAKTURA, чёрный + дерево, плавные градиенты — см. memory). Произвольная настройка цветов/контуров блоков легко сломает визуальную целостность.
- Если действительно нужно — могу добавить ограниченный набор настроек на секцию (фон, цвет акцента из палитры, вертикальные отступы S/M/L), без полного «конструктора».
