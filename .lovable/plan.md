# План правок

Работаем **строго по одному этапу за раз**. После каждого этапа — короткий отчёт и «ок, дальше».

---

## Этап 1 — письма админу о новых заявках

**Что сейчас:** заказы уже шлют письмо админу (`order-place` → `renderAdminNewOrder` → `ADMIN_NOTIFY_EMAIL`). А заявки из формы «Свяжитесь с нами» пишутся напрямую в таблицу `contact_requests` из браузера — админ не получает никакого письма.

**Что делаю:**
1. Новая edge-функция `contact-submit` (public): валидация полей → insert в `contact_requests` → отправка письма админу через уже существующий `renderContactRequest` из `_shared/email.ts`.
2. `src/components/ContactForm.tsx` перевожу с прямого `supabase.from('contact_requests').insert` на вызов `contact-submit`.
3. Логи писем уже пишутся в `email_log` — ничего дополнительно не нужно.

**Проверка:** отправить тестовую заявку с сайта → в почте админа лежит письмо, в админке в «Письмах» — запись `sent`.

**Что НЕ делаю:** не трогаю заказы (там уже работает), не меняю шаблон письма, не создаю новых секретов — используем существующий `ADMIN_NOTIFY_EMAIL`/`SMTP_*`.

## Этап 2 — единый источник категорий (главная = каталог = меню)

**Что сейчас:** три несвязанных источника.
- Главная (`CategoriesSection`) — читает из `app_settings` (редактируется в «Контент → Главная»).
- Каталог (`CatalogPage`) — берёт хардкод `categories` из `src/data/products.ts` + локальные картинки.
- Меню в шапке (`Header`) — тоже свой список.

Добавляешь категорию на главную — в каталоге и меню её нет.

**Что делаю:**
1. Миграция: новая таблица `product_categories` (`slug`, `name`, `description`, `image_url`, `sort_order`, `show_on_home`, `show_in_menu`, `is_active`, timestamps). GRANTs + RLS (public SELECT активных, admin через service_role). Сидирую текущими категориями из `src/data/products.ts` + текущими картинками (URL из бакета либо переносим ассеты).
2. Подкатегории пока оставляю в `src/data/products.ts` (это отдельная задача, если понадобится — вынесем в `product_subcategories` в следующем плане). Категории верхнего уровня — единый источник.
3. Хук `useCategories()` (React Query, кеш) читает активные категории; в фильтрах — `show_in_menu`, на главной — `show_on_home`.
4. `CategoriesSection`, `CatalogPage`, `Header`, `CatalogFilters` переводятся на `useCategories()`. Удаляю дублирующий раздел «Категории на главной» из «Контент → Главная», вместо него — переход в новый раздел.
5. В админке — новый раздел «Категории каталога» (уже был в старом плане п.11, теперь делаем полноценно): CRUD, загрузка картинки, `show_on_home`/`show_in_menu`/`is_active`, порядок сортировки drag-n-drop.

**Проверка:** добавил категорию с флагами `show_on_home` и `show_in_menu` → появилась на главной, в каталоге среди плиток и в меню шапки.

**Что НЕ делаю:** не трогаю подкатегории, не мигрирую поле `products.category` — оно остаётся строковым slug, совпадающим со slug в `product_categories`.

## Этап 3 — реструктуризация раздела «Контент сайта» в админке

**Что сейчас:** вкладка «Контент» — одна огромная простыня, всё вперемешку, приходится скроллить, порядок неочевиден.

**Что делаю:** внутри вкладки «Контент» — вложенные под-вкладки в порядке меню сайта:

1. **Главная** — hero (видео/заголовок), популярные (настройки блока), преимущества, порядок блоков.
2. **Каталог** — заголовок, описание, SEO. (Сами категории живут в отдельном разделе «Категории каталога» после этапа 2 — здесь просто ссылка на него.)
3. **Услуги** — текущий редактор `services_page`.
4. **Доставка и оплата** — текущий редактор `delivery_page`.
5. **Галерея** — заголовок/описание страницы (сами картинки — в разделе «Галерея»).
6. **Блог** — заголовок/описание страницы (посты — в разделе «Блог»).
7. **Контакты** — адрес, телефоны, карта, соцсети.
8. **SEO** — текущий редактор мета-тегов по страницам.
9. **Меню и футер** — навигация, футер.

Каждая под-вкладка:
- Компактный layout, поля сгруппированы карточками с заголовками.
- Над каждой нетривиальной секцией — короткий help-текст курсивом («Что это и где отображается»).
- Кнопка «Сохранить» на каждой под-вкладке отдельная, sticky снизу.
- Ничего не удаляю из существующих настроек — просто перекладываю по местам.

**Технически:** выношу «Контент» из `AdminPage.tsx` в `src/pages/admin/content/` — по одному файлу на под-вкладку + `ContentTabs.tsx` роутер. Это единственная структурная правка, чтобы файл админки не рос ещё на 3000 строк.

**Проверка:** захожу в «Контент» → вижу под-вкладки в порядке меню → каждая под-вкладка помещается в экран без километрового скролла → все существующие поля на месте, сохраняются.

**Что НЕ делаю:** не переделываю дизайн админки в целом, не трогаю остальные разделы (Товары/Заказы/…), не меняю схему данных `app_settings`.

---

## Порядок

Предлагаю идти **1 → 2 → 3**. Этап 1 — быстрый и независимый. Этап 2 — средний, миграция + переключение трёх мест. Этап 3 — самый объёмный по коду, но чисто фронтенд-рефакторинг админки.

Скажи «ок, этап 1» — стартую.
